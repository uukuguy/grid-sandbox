# octo-workbench v1.0 发布冲刺设计方案

> 日期：2026-03-04
> 状态：已确认，进入实施
> 目标：面向开发者/技术团队的企业级通用自主智能体，单阶段完整交付

---

## 一、产品定位

**核心定位**：企业通用任务为主、兼顾代码工程的自主智能体，单用户单实例部署。

**三种交互模式**：
1. **对话模式**——实时流式响应，工具调用可见，支持取消
2. **后台任务模式**——异步提交，后台自主执行，查询结果
3. **定时调度模式**——Cron 触发，任务队列管理，执行历史

**明确不在 v1.0 范围**：多用户/多租户、Multi-Agent 编排、浏览器自动化（可 MCP 接入）、语音输入输出。

## 四、架构关键调整

### 调整1：ToolRegistry 版本化共享引用（解决 S2）

```rust
// 调整前
AgentRuntime.tools: Arc<StdMutex<ToolRegistry>>
start_primary() → build_tool_registry() → Arc::new(快照) → AgentExecutor

// 调整后
AgentRuntime.tools: Arc<RwLock<ToolRegistry>>
AgentRuntime.tools_version: AtomicU64

AgentExecutor.tools: Arc<RwLock<ToolRegistry>>   // 持有共享引用，非快照
AgentExecutor.tools_version: u64                  // 上次使用的版本

// AgentLoop 每 round 前检查版本，按需重建只读快照
if executor.tools_version != runtime.tools_version.load() {
    重建 round 内快照
}
```

**影响范围**：`runtime.rs`、`executor.rs`、`loop_.rs`

### 调整2：AgentExecutor 生命周期正确终止（解决 S1）

```rust
// 调整前
stop_primary() → send(Cancel) → cancel_flag=true，while 继续

// 调整后
stop_primary() → drop(AgentExecutorHandle)
              → tx drop → rx.recv() 返回 None
              → while 自然退出 → tokio task 结束

// AppState.agent_handle 改为 Option<AgentExecutorHandle>
// stop 时 take() 触发 drop
```

**影响范围**：`runtime.rs`、`state.rs`

### 调整3：后台任务 API（实现 C3，修复 S3）

```
POST /api/tasks     → 创建任务 → 直接调用 execute_scheduled_task()
GET  /api/tasks     → 任务列表（status: pending/running/done/failed）
GET  /api/tasks/{id} → 任务详情和结果
DELETE /api/tasks/{id} → 取消任务
```

**影响范围**：`scheduler/mod.rs`（修复 run_now）、新增 `api/tasks.rs`

### 调整4：WorkingMemory per-session 隔离（解决 S4）

```rust
// start_primary() 中：
// 调整前：self.memory.clone()（全局共享）
// 调整后：Arc::new(InMemoryWorkingMemory::new())（每 session 独立）
```

**影响范围**：`runtime.rs` 约 5 行
