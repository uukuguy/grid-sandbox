## Reviewer: Claude (current session, fresh subagent)

### Question 1: Audit framework completeness
**Verdict**: GAPS-FOUND

**Findings**:
- 8 audit points are not 8 distinct dimensions. §P5 reduces to ~2 independent dimensions once §F is admitted as a "specification" of §C.1 (per cross-ref counts at PLAN L805-810). Framework presents 8 questions but probes ~5–6.
- Hidden assumption in §P5.1: "stagnation/pricing unfit" is undefined and `4.1-PRE-AUDIT-NOTES.md:73-78` already collapses external-vs-self axis to "semantically weakened" — no operational definition of stagnation when EAASP is self-developed. Plan locks `partial (语义弱化)` (PLAN:288-300) without forcing operationalization.
- §P5.4 "team surplus ≥30%" — `4.1-PRE-AUDIT-NOTES.md:78` hardcodes "default 成立" before audit runs. Plan blesses `partial (baseline 默认成立)` (PLAN:466-474) and routes actual percentage to Open Items. Forecasting question masquerading as verdict.
- **Missing dimensions a CTO would need**:
  - Sunk cost in Leg B scaffolding (paper-crate decay rate per ADR-V2-023 R1)
  - Opportunity cost (what does NOT get done if 两腿都推?)
  - Customer/partner dependencies on EAASP fork timing
  - Technical-debt accumulation rate (D109/D134/D136/D142/D143 cross-milestone watchlist)
  - Regulatory/compliance exposure beyond Q3 (国密/WORM/信创 timelines from skill-spec-coverage doc)
- §P5 vs §F redundancy: §F is essentially Q1 specialization of §C.1; §P5 was built on different conceptual axis (product form). Audit forces them to "cross-reference" but never asks whether they jointly cover the decision space.

**Top recommendation**: Add §3.5 audit point "Decision-space coverage check" with explicit verdicts on **5 missing dimensions** — (a) Leg B carrying cost / paper-crate decay rate, (b) opportunity cost, (c) external commitment dependencies, (d) cross-milestone debt rate, (e) regulatory deadline pressure. Each verdict can be `unknown` and feed Open Items, but their absence from framework is the leak.

### Question 2: Evidence standard rigor
**Verdict**: NEEDS-TIGHTENING

**Findings**:
- "User direct statement" tier (D-E-01) has zero protection against confirmation bias when user is also decision-maker. `4.1-PRE-AUDIT-NOTES.md:108-114` lists 5 baseline conclusions pre-approved without evidence requirement. §D Sanity Guards only fire when audit **contradicts** baseline (asymmetric falsifiability).
- Open Items backstop is weak: D-E-04 says unknowns go to "待 user 补 by Phase 4.2 启动 ADR-V2-024 Accepted 前" — no rule like "if N of 5 unfilled, Phase 4.2 cannot proceed".
- Forecasting questions inadequately served by 4-string verdict vocabulary. §P5.4 (team capacity over time) and §F.Q1 (vertical roadmap commit) are probability/scenario questions. Plan locks D-E-03 to exactly 4 strings and rejects synonyms via grep — actually **prevents** audit from expressing forecast confidence.
- 三选一 binary shape vs. nuanced gradient: framing is "走 Leg A 硬化 / 走 Leg B 激活 / 两腿都推进" enforced verbatim. "两腿都推进" is then secretly used as low-resolution smear including "engine vs data/integration" double-axis. Vocabulary forces audit to claim it picked from 三选一 when it actually proposes a 4th option in §5.

**Top recommendation**: Add a 5th verdict value `forecast` with required structured payload (scenario A: prob X / scenario B: prob Y / driver / horizon) for §P5.4, §F.Q1, §F.Q2. Add hard rule in D-E-04: "If Open Items >2 unfilled at Phase 4.2 start, `partial (baseline 默认成立)` verdicts auto-downgrade to `unknown`, blocking ADR-V2-024 Accepted."

### Question 3: Bias / scope risks
**Verdict**: CONCERN

**Findings**:
- D-B-05 "audit 不**否决** §P5 4 条原文" is methodologically compromised. If §P5 framing is wrong (e.g., presupposing Leg B is binary opposite of Leg A while user's actual model per §C.1 is "engine vs data/integration"), "partial weakening only" prevents audit from saying so cleanly.
- §5 framework revision is **buried**, not surfaced. Plan locks SC#2 to 三选一 verbatim; §5 sits as "附加观察" with explicit "decision power 在 Phase 4.2" (D-F-05). Reader of §1 Summary sees "三选一推荐: 两腿都推进" with framework-revision listed beside it, but verification only enforces 三选一 verbatim.
- Enforcing exact 三选一 wording via grep risks "closest fit" picking. Plan reaches for "两腿都推进" (broadest of three) and lets audit avoid hard call. 4th option ("neither — pivot to engine/data-integration framing") would be substantively correct per §C.1 but cannot be SC#2 verdict.
- Sanity Guard asymmetry: 5 baseline conclusions pre-confirmed, 5 contradictory require "硬证据". Auditor suspecting "user's 工时 self-report is off vs git log" has no protocol entry.
- D-C-02 leverage: framework revision as "附加 observation" is structurally weaker than "alternative recommendation". Phase 4.2 ADR-V2-024 §Decision will not be required to engage §5's substance.

**Top recommendation**: Replace D-B-05 with "audit MAY override §P5 framing with a written supersession warrant; warrant requires structured rebuttal of each §P5 trigger and an alternative-frame proposal that survives §D Sanity Guards." Drop SC#2's verbatim grep; require the recommendation to be a single labeled option from {Leg A, Leg B, both, framework-revision, hybrid-with-warrant}.
