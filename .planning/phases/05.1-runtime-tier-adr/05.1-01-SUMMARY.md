## Decisions Made

- **Decoupled contract execution strength from product track.** ADR-V2-017 三轨 (主力 + 样板 + 对比) is a product strategy (who maintains what); ADR-V2-025 四档 (主力/样板/参考/冻结) is a contract strategy (who must PASS what). They are orthogonal — a runtime can be in 样板产品轨 AND 主力契约档 (none currently is, but the model allows it). This avoids overloading "ADR-V2-017 主力" with a contract-execution meaning that conflates "we maintain it" with "v1.x must PASS on it".
- **Hermes Frozen tier gets explicit规范出口.** Rather than letting hermes inherit `xfail: true` and clutter CI runs, the matrix carries `xfail: 'skip'` for hermes and an early-return step short-circuits all subsequent build/test steps. This preserves ADR-V2-017 §冻结决议 verbatim (no policy change to hermes) while giving CI a clean skip signal in the PR Checks panel.
- **Parametrize-then-lazily-resolve preserves single-launch.** Pytest parametrize over `runtime_name` would normally create 7 session-cache keys for the chained `runtime_config` → `runtime_launcher` → `runtime_grpc_stub` fixtures, potentially launching 7 subprocesses if all 7 cases ran. The skip guard fires BEFORE `getfixturevalue`, so 6 of 7 cases skip without ever resolving the chain. Only the case matching `--runtime=<one>` resolves and launches. Net: zero behavior change in CI subprocess count, full collection visibility locally.
- **NEW- namespace separation from EAASP D#.** Phase 4a project review introduced NEW-D2, NEW-E2, NEW-E3, NEW-C1, NEW-C2, NEW-C3 (per STATE.md Deferred Items table). These are valid Deferred items but in a separate ID space than the EAASP D# (D1–D155). Adding NEW-D2 to the EAASP closed-list count would distort the historical 38-closed-of-68-total stat. Instead, the 统计汇总 table grew a parallel row for "✅ closed (NEW- 命名空间)" with an explicit note. Future Phase 5+ NEW- closures slot into the same row.
