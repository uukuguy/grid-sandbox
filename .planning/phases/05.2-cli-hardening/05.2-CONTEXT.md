# Phase 5.2: CLI Hardening - Context

**Gathered:** 2026-05-02
**Status:** Ready for planning

<domain>
## Phase Boundary

`grid-cli` (binary `grid`) 用户面硬化 — 6 个 requirement 范围 (CLI-01..06):
1. 命令树 taxonomy + verb-noun 嵌套
2. ChunkType streaming 渲染 UX
3. Error message + sysexits.h exit code 一致性
4. TUI `key_handler.rs` 拆分 (NEW-C2 deferred 项, 1556 LOC → <300 LOC/file)
5. Session lifecycle (`session list / resume / kill`) 端到端打磨
6. `grid doctor` 检查清单扩展

**不在 scope:** 任何新 capability (e.g., `grid skill publish`, multi-tenant session sharing, web UI parity), grid-server 改动, contract evolution (Phase 5.3 owns), proto 改动, live LLM e2e。

</domain>

<decisions>
## Implementation Decisions

### CLI-01 — 命令树 Taxonomy

- **D-01**: 采用业界标配两层 verb-noun 嵌套 (git/docker/gh 风格), `grid <noun> <verb>` 结构。
- **D-02**: 拆 `grid-studio` 出独立 binary。`grid` 主 binary 只管 cli-ask / session / config / doctor / mcp / memory / tools / skill 这类无状态命令; `grid-studio tui` / `grid-studio dashboard` 走独立 binary, feature 默认关闭, 需要 `cargo install --features studio` 才装。理由: 双轴模型下 grid-cli 核心定位是 substitutable L1 runtime 客户端, studio 是开发者工具, EAASP test harness 永远不会调 studio; 拆出去也让 Phase 5.4 grid-server (axum+tower) 依赖图与 grid-cli 解耦。
- **D-03**: `cli-ask` → `ask` 等改名走 deprecation alias 路径, 保留旧名 + stderr 第一行输出 `warning: 'cli-ask' is deprecated, use 'ask' instead (will be removed when no in-tree consumers remain)`。
- **D-04**: 旧别名删除时机 = **grep 归零**, 不绑 milestone。判定方式: `grep -r "grid cli-" Makefile scripts/ examples/ tests/` 全部归零后任意 phase 顺手删。这是 Grid 没有外部下游消费者的现实 — "代码里没人用了就删" 比按 milestone 删更不黑盒。
- **D-05**: `grid --help` 命令分组采用 5 组分类:
  ```
  INTERACT      ask, run, agent
  SESSION       session (list/resume/kill), state
  CONFIG        config, init, completions
  DIAGNOSTIC    doctor, dashboard-cert, dashboard-security, dashboard-auth
  RESOURCES     mcp, memory, tools, skill, sandbox, eval
  AUTH          auth
  ```
  `dashboard*` 三个 (dashboard / dashboard-cert / dashboard-security / dashboard-auth) 并入 DIAGNOSTIC, 不单独成组。
- **D-06**: CI gate — 落地 `docs/design/Grid/cli-rename-migration.md` 跟踪表; 加一条 `make` target 或 CI step 扫描 in-tree 消费者引用 (Makefile + scripts/ + examples/ + tests/ + lang/ + tools/), 输出未迁移调用的清单作为 deprecation tracking source-of-truth。

### CLI-02 — ChunkType Streaming 渲染 UX

- **D-07**: 业界标配整体接受 (typewriter / collapse tools / dim continuation / NO_COLOR + `--no-color` flag), 但 typewriter **默认关闭, TTY 自动检测开 + `--stream` flag 显式开**。这是偏离业界标配的一处 — 业界 (claude-code / cursor) 默认开 typewriter, 但 Grid 主消费场景之一是 EAASP test harness / Makefile / pytest pipe stdout, typewriter delay 就是污染。"默认关 + 显式开" 比 "默认开 + auto-detect 关" 更安全, 且和 Grid 作为 substitutable runtime (CLI 输出消费者很多时候是另一个程序) 的定位一致。
- **D-08**: TOOL_START / TOOL_RESULT collapse-by-default, `--verbose` 展开。
- **D-09**: WORKFLOW_CONTINUATION 显示为 dim grey 内联 marker (`(continuing…)`), 不换行不隐藏。
- **D-10**: ERROR chunk 红色高亮 + 单行内联 (不做 sticky footer)。
- **D-11**: TTY-not-detected → 自动 `--output text`; respect `NO_COLOR` env (per https://no-color.org/) 与 `--no-color` flag (CLI flag 优先级高于 env)。
- **D-12**: 所有 ChunkType render path 必须支持 `--output {text,json,stream-json}` 三档不变 (现有 `output/` 模块基础)。

### CLI-03 — Error & Exit Code 一致性

- **D-13**: 业界标配整体接受 — sysexits.h 映射如下 (与 PLAN T3 已列项一致, 锁定):
  - EX_USAGE = 64 (CLI argument 错误, missing required flag)
  - EX_NOINPUT = 66 (config file 缺失 / 不可读)
  - EX_NOPERM = 77 (API key invalid / auth 失败)
  - EX_UNAVAILABLE = 69 (provider 错误 / network timeout / runtime gRPC 不可达)
  - EX_SOFTWARE = 70 (内部 bug / unexpected error)
  - EX_CONFIG = 78 (config 内容错 / schema mismatch)
- **D-14**: 错误格式 = 单行 + cause chain 通过 `--verbose` 展开。"Suggestion" 行不内置 fix-suggestion 表 — Grid 走 Unix 风格 (`ls` / `cat`) 而非 "fail with hint" 工具风格 (`cargo` / `gh`)。
- **D-15**: CLI 层不再做 retry — grid-engine 已经有 provider retry, CLI 不重复。

### CLI-04 — TUI `key_handler.rs` 拆分

- **D-16**: 模块切分维度 = **按状态切** (REPL-conversational TUI 的 OverlayMode + 跨切 sub-state, 各一个文件), 不按动作切。理由: ratatui 生态 + helix / zellij 类 TUI 都按状态/模式切; Grid TUI 是 REPL-conversational 风格 (与 `tui/app_state.rs` 的 `OverlayMode` enum + 跨切 sub-state 对齐, 不是 vim editor 风格)。
- **D-17**: Dispatcher 用 `match` on `(overlay_mode, sub_state, key)` 单文件实现, 转发到状态模块。不用 `trait KeyHandler` + dyn dispatch (在 ≤8 状态场景下是 over-engineering, 且 match 让"哪个状态处理哪些键"在一个文件里一眼看完)。
- **D-18**: 文件结构 (按代码实际状态命名, **不是** vim editor mode 命名 — 修订自 2026-05-02 research, 原假设 Normal/Insert/Command/Visual/Help/Dialog 与 codebase 不匹配):
  ```
  tui/key_handler/
    mod.rs          # dispatcher (match on (overlay_mode, sub_state, key))
    normal.rs       # OverlayMode::None 主 chat input 路径 (默认状态)
    vim.rs          # vim mode toggle 子状态 (when input box vim_mode on)
    search.rs       # history_search 子状态 (Ctrl+R 等触发)
    selector.rs     # model_selector / SessionPicker 子状态
    overlay.rs      # AgentDebug / Eval overlay (OverlayMode::AgentDebug | Eval)
    approval.rs     # pending_approval 阻塞模态
    common.rs       # 跨态共享逻辑 (Esc / Ctrl+C / 全局快捷键)
  ```
  > **修订原因 (2026-05-02 research)**: discuss-phase 假设 vim editor mode 拓扑 (Normal/Insert/Command/Visual/Help/Dialog), 读代码后发现实际是 REPL-conversational TUI — 主 overlay 维度是 `OverlayMode { None, AgentDebug, Eval, SessionPicker }` + 跨切 sub-state (vim_mode toggle / history_search / model_selector / pending_approval), 与 vim editor 拓扑差异较大。按代码实际命名以避免 cognitive dissonance。D-16 的 "按模式切 vs 按动作切" 决策不变, 只是 mode 名称从 vim 风格改为 REPL 风格。
- **D-19**: Invariant 表 (拆分前 source-of-truth 防 silent regression):
  - 第一个 commit: 在现有 `key_handler.rs` 顶部加 ≤200 行注释表, schema 用轻量版 `(Mode, Key) → Action` (不要 pre/post-condition 列)
  - 拆完最后一个 commit: 搬进 `tui/key_handler/INVARIANTS.md` (或 README.md)
- **D-20**: 测试策略:
  - **单元测试** (在每个状态文件 `#[cfg(test)] mod tests` 内): 每个状态 ≥3 个 test (happy path + 1 edge case + Esc/Ctrl+C 行为), 7 状态 × 3 = ≥21 tests, 大幅超过 SC#4 ≥6 要求
  - **Integration test** (在 `crates/grid-cli/tests/key_handler_integration.rs`): 跨状态序列 ≥2 个 (e.g., `Normal → Ctrl+R → search query → Esc → Normal` / `Normal → AgentDebug overlay → Esc → Normal`)
  - **不用** ratatui `TestBackend` 跑全 TUI render assertion (重, 模拟 KeyEvent 直接灌 `dispatch(overlay_mode, sub_state, key)` 断言 Action enum 输出即可)
- **D-21**: 文件长度目标: 拆完每个文件 <300 LOC (per ROADMAP SC#4); `mod.rs` dispatcher ≤150 LOC。

### CLI-05 — Session Lifecycle (the agent's Discretion)

- **D-22**: the agent's discretion 范围内做 sane defaults:
  - `session list`: 列 (id, started_at, last_turn_at, message_count, model), 默认按 `last_turn_at` 倒序 (最近优先), `--sort created|recent|model` 可选
  - `session resume`: 默认从 last turn + 1 继续, 不重放; 首次 resume 显示 "Resuming session {id}, {N} turns since {time}. Press Ctrl+C to abort."
  - `session kill`: 默认 soft-delete (SQLite row 标 `deleted_at`), `--purge` flag 硬删
  - Orphan 阈值 = 24h 无 turn, `session list` 给 orphan row 加灰色标记, `session prune` 命令手动清理 (不 auto-prune)
- **D-23**: 这些 default 写完跑过一遍后用户用了再调, 不锁。

### CLI-06 — `grid doctor` 检查清单

- **D-24**: ROADMAP 列的 5 条 must-check 全要 (EAASP_DEPLOYMENT_MODE / GRID_HOOKS_FILE / hook bundle 健康度 / L2 memory engine 可达性 / L1 runtime gRPC 可达性)。
- **D-25**: **加一条 Grid 独有 check: `proto-cli-sync`** — 读 `proto/eaasp/runtime/v2/runtime.proto` ChunkType enum 全集, 对比 CLI render 路径里 (`crates/grid-cli/src/output/text.rs` 或 `_ALLOWED_CHUNK_TYPES` whitelist) 的 ChunkType arm 全集, diff 直接报。≤40 LOC。理由: 双轴模型下 contract evolution 在 Phase 5.3 做 (CONTRACT-01 ChunkType 扩展), 5.2 doctor 顺手把这个 lint 钉上 → 5.3 加新 ChunkType 时如果忘了同步 CLI render, doctor 会 fail 提醒, 在 contract test 跑之前抓住。
- **D-26**: 输出格式: 默认 tabular `name | status | detail`, `--json` flag 切结构化输出 (for CI consumption)。
- **D-27**: 严重度: 三档 — `error` (exit 1, blocks CI gate)、`warn` (exit 0, 显示但不阻断)、`info` (绿色 OK 行)。EAASP_DEPLOYMENT_MODE 缺失为 `warn` (per ADR-V2-019 仍 Proposed); proto-cli-sync mismatch 为 `error`; gRPC reachability 失败为 `warn` (因为 doctor 可能离线跑)。
- **D-28**: 网络 check 加 `--offline` flag 跳过, 默认 timeout 5s/check。
- **D-29**: **不做** `--fix` auto-fix 模式 (strictly read-only)。

### the agent's Discretion

- CLI-02 子细节: ANSI 颜色具体值, dim grey 怎么实现 (crossterm 库特性); typewriter 速度 (default 30ms/char vs 50ms/char); markdown 内联渲染 (`**bold**` / `code` 等) 是否走 `ui/markdown.rs` 现有路径 — researcher / planner 决定。
- CLI-04 mod.rs 内部 enum 命名 (`Action::EnterInsertMode` vs `Action::EnterMode(Mode::Insert)`); pub vs pub(super) 边界 — researcher / planner 决定。
- CLI-05 SQLite schema 演进 (新增 `deleted_at` 列): 是否新 migration 文件 vs `tokio-rusqlite` 启动时 ALTER — researcher / planner 决定。
- CLI-06 `grid doctor --json` schema (字段名: `name` / `status` / `detail` / `severity` 等) — researcher / planner 决定。

## Existing Code Insights

### Reusable Assets

- `crates/grid-cli/src/output/` 三档 (text/json/stream_json) 已成熟, CLI-02 是改 text/stream_json 内部渲染逻辑, 不增 output 模式
- `crates/grid-cli/src/commands/doctor.rs` 已存在, CLI-06 是扩展 check 列表, 不是新建
- `crates/grid-cli/src/commands/session.rs` 已存在, CLI-05 是补强 list/resume/kill subcommand 内部实现, 不是新建
- `crates/grid-cli/src/tui/app_state.rs` 的 Mode enum 是 key_handler 拆分的 source-of-truth, 不需要改
- `crates/grid-cli/src/repl/slash.rs` 已有 slash command 解析, REPL 模式的 Command mode handler 可以复用
- `clap` 是现有依赖, verb-noun 嵌套用 `Subcommand` derive 即可, 无需新依赖

### Established Patterns

- **`anyhow::Result<()>` 在 main**: error 通过 `anyhow::Error` 链, exit code 由 main 包装 → CLI-03 sysexits 映射在 main.rs 末尾的 `Err(e) => { exit_code(&e); std::process::exit(code); }` 单点收口
- **`tokio::main`**: 异步 runtime 已经在 main.rs, CLI-03 retry 决策 (CLI 不再做 retry) 与 grid-engine 的 RetryPolicy::graduated 已实现一致
- **`tracing` + `init_logging(verbose: bool)`**: 已有 verbose flag, CLI-03 cause chain 通过 `tracing::error!(error = ?e, "...")` 输出
- **`pub fn` 单点收口**: commands/*.rs 每个文件 expose 单一 `pub async fn run(...)` 入口, mod.rs aggregate, dispatcher 在 main.rs match arg → call run

### Integration Points

- **grid-cli ↔ grid-runtime gRPC**: `grid doctor` 的 L1 runtime reachability check 走 16-method RuntimeService 的 Initialize() ping (timeout 5s), 不实际 SendResponse
- **grid-cli ↔ grid-engine**: `grid ask` 流式输出 ChunkType 路径走 grid-engine harness, CLI-02 是消费层渲染, 不改 engine
- **grid-cli ↔ tools/eaasp-l2-memory-engine**: `grid doctor` 的 L2 memory reachability check 走 MCP `memory_search` tool ping, 不实际写入
- **grid-cli ↔ SQLite**: session lifecycle 走 `data/grid.db` (path from `GRID_DB_PATH`), `tokio-rusqlite` 已经是依赖, CLI-05 schema migration 复用现有 migration 框架

<specifics>
## Specific Ideas

- **业界标配 = 主基线**: git/docker/gh 风格的 verb-noun 嵌套, claude-code/aider/opencode 风格的 streaming 渲染, sysexits.h 风格的 exit code, Unix 风格的 fail-without-suggestion 错误格式 — 用户明确表态 "业界标配可以"。
- **偏离业界标配的两处** (用户明确同意):
  1. CLI-02 typewriter **默认关闭** (业界默认开), 因为 Grid 主消费场景之一是 EAASP test harness / Makefile pipe stdout, typewriter delay 是污染。
  2. CLI-01 `cli-` 前缀别名 **删除时机绑 grep 归零**, 不绑 milestone (因为 Grid 没有外部下游消费者)。
- **新增的 Grid 独有补强** (用户明确同意):
  3. CLI-04 拆分前先写 invariant 表 (`(Mode, Key) → Action`), 防 silent regression — NEW-C2 已 defer 一次, 这次拆完不能再有 silent bug 让用户骂自己。
  4. CLI-06 `grid doctor` 加 `proto-cli-sync` check — 双轴模型下 Phase 5.3 contract evolution 加新 ChunkType 时, doctor 自己会 fail 提醒, 是 5.3 升级的前置健康度。

</specifics>
