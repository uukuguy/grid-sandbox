# Phase 5.2 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.

> **Source:** Section §9 of `05.2-RESEARCH.md` (Validation Architecture). This file consolidates the modality table for Nyquist Dimension 8 enforcement during planning.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | `cargo test` (Rust workspace) + `python3 -m pytest` (proto-cli-sync helper, optional) |
| **Config file** | `Cargo.toml` workspace; `crates/grid-cli/Cargo.toml` for crate-level deps |
| **Quick run command** | `cargo test -p grid-cli --lib --quiet` (per-task; ~10-20s) |
| **Full suite command** | `cargo test -p grid-cli --quiet` (lib + integration; ~30-60s) |
| **Estimated runtime** | ~30-60s for full grid-cli; per-task 10-20s |

**Test discipline (per CLAUDE.md):** never run full workspace test (`cargo test --workspace`) unsolicited — only `-p grid-cli` scope for this phase.

---

## Sampling Rate

- **After every task commit:** Run `cargo check -p grid-cli` (≤10s) + `cargo test -p grid-cli --lib --quiet` for the touched module path
- **After every plan wave:** Run `cargo test -p grid-cli --quiet` (full grid-cli, ~30-60s)
- **Before `/gsd-verify-work`:** `cargo test -p grid-cli` green; targeted manual `grid --help` snapshot inspection
- **Max feedback latency:** ≤60s (well within 5-min Nyquist budget)
